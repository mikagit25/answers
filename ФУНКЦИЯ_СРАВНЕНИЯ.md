# 🆕 Функция сравнения традиций - Теперь работает!

## ✅ Что добавлено

Теперь после получения ответа от одной традиции, вы можете нажать кнопку **"🔍 Посмотреть, что говорят другие традиции"** и система покажет как этот же вопрос отвечают 3 другие религиозные/философские традиции.

---

## 🎯 Как это работает

### Шаг 1: Задайте вопрос
Выберите традицию (например, "Стоицизм") и задайте вопрос.

### Шаг 2: Получите ответ
Система генерирует ответ от выбранной традиции через Groq API.

### Шаг 3: Нажмите кнопку сравнения
Под ответом появится розовая кнопка:
```
🔍 Посмотреть, что говорят другие традиции
```

### Шаг 4: Увидьте сравнение
Система автоматически:
1. Исключает текущую традицию из списка
2. Выбирает 3 другие традиции
3. Генерирует краткие ответы (до 60 слов) от каждой
4. Показывает их в красивых карточках

---

## 📊 Пример использования

**Вопрос:** "Как преодолеть страх перед будущим?"

**Основная традиция:** Стоицизм
→ Получаете развернутый ответ о стоическом подходе

**Нажимаете кнопку сравнения** → Видите:

1. **Христианство** 
   - Ответ о доверии Богу и молитве

2. **Ислам**
   - Ответ о таваккуле (уповании на Аллаха)

3. **Буддизм**
   - Ответ о принятии непостоянства и осознанности

---

## 🔧 Технические детали

### API Endpoint
```bash
POST /api/v1/compare
Content-Type: application/json

{
  "question": "Как преодолеть страх?",
  "exclude": "stoicism"
}
```

### Response
```json
{
  "comparisons": [
    {
      "tradition_id": "christianity",
      "tradition_name": "Христианство",
      "summary": "Краткий ответ (до 60 слов)...",
      "source_preview": "Ответ основан на принципах Христианства"
    },
    // ... еще 2 традиции
  ],
  "question": "Как преодолеть страх?",
  "excluded_tradition": "stoicism"
}
```

## 💡 Особенности реализации

### Автоматический выбор традиций
- Система исключает текущую традицию
- Выбирает следующие 3 из списка
- Порядок: Христианство → Ислам → Буддизм → Иудаизм → Индуизм → Даосизм → Гуманизм

### Оптимизация запросов
- Каждый ответ ограничен 150 токенами (~60 слов)
- Используется тот же LLM менеджер с fallback
- Параллельные запросы к Groq для скорости

### Обработка ошибок
- Если одна традиция не отвечает, показывается заглушка
- Другие традиции продолжают загружаться
- Пользователь видит частичный результат

## ⚡ Производительность

**Время ответа:**
- Основной ответ: ~2 секунды (Groq)
- Сравнение (3 традиции): ~5-7 секунд
- Итого: ~7-9 секунд для полного ответа

**Оптимизация:**
- Groq очень быстрый (<1 сек на ответ)
- Запросы выполняются последовательно
- Можно улучшить параллельным выполнением
