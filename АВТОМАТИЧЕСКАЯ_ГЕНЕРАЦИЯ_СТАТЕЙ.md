# 🤖 Автоматическая генерация SEO-статей

## 📊 Обзор системы

Система автоматически собирает пользовательские вопросы, анализирует популярные темы и генерирует полноценные SEO-статьи с помощью LLM.

---

## 🎯 Как это работает

### Шаг 1: Сбор данных

**Каждый раз когда пользователь задает вопрос:**
```python
POST /api/v1/ask
{
  "question": "Как справиться со страхом?",
  "tradition_id": "stoicism",
  "language": "ru"
}
```

**Система автоматически:**
1. ✅ Сохраняет вопрос в базу (`data/user_questions.json`)
2. ✅ Классифицирует тему (fear_anxiety, meaning_life, relationships, etc.)
3. ✅ Извлекает ключевые слова
4. ✅ Отслеживает количество вопросов по каждой теме
5. ✅ Запоминает какие традиции отвечали

### Шаг 2: Анализ популярных тем

**Когда накоплено достаточно вопросов:**
```bash
GET /api/v1/articles/popular-topics?min_questions=3
```

**Возвращает:**
```json
{
  "popular_topics": [
    {
      "topic": "fear_anxiety",
      "question_count": 15,
      "traditions": ["stoicism", "buddhism", "christianity"],
      "sample_questions": [
        "Как преодолеть страх перед экзаменом?",
        "Что делать при панической атаке?",
        "Как перестать бояться будущего?"
      ],
      "avg_word_count": 350
    }
  ]
}
```

### Шаг 3: Генерация статьи

**Ручная генерация:**
```bash
POST /api/v1/articles/generate?topic=fear_anxiety&language=ru
```

**Или массовая генерация:**
```bash
POST /api/v1/articles/generate-all?language=ru
```

**Процесс генерации:**
1. 📝 LLM анализирует все вопросы по теме
2. 📚 Сравнивает подходы разных традиций
3. ✍️ Пишет статью 2000+ слов с:
   - Введением
   - Историческим контекстом
   - Подробным описанием подходов каждой традиции
   - Сравнительным анализом
   - Практическими рекомендациями
   - Реальными примерами
   - FAQ секцией
   - Заключением
4. 🏗️ Создает HTML файл с SEO разметкой
5. 🔗 Обновляет sitemap.xml
6. 💾 Сохраняет метаданные о статье

## 🎨 Шаблон статьи

Каждая сгенерированная статья включает:

### 1. SEO Meta-теги
```html
<title>Как преодолеть страх и тревогу: мудрость мировых традиций</title>
<meta name="description" content="Полное руководство...">
<meta property="og:type" content="article">
```

### 2. Schema.org разметка
```json
{
  "@type": "Article",
  "headline": "Как преодолеть страх...",
  "author": {"@type": "Organization", "name": "Answers Platform"}
}
```

### 3. Структура контента
- **H1**: Заголовок статьи
- **H2**: Основные разделы (8-10 разделов)
- **H3**: Подразделы
- Списки, цитаты, выделения
- **FAQ секция** с микроразметкой
- Footer с ссылкой на главную

### 4. Оптимизация для поисковиков
- Уникальный title (50-60 символов)
- Уникальное description (150-160 символов)
- Ключевые слова в заголовках
- Внутренние ссылки
- Mobile-friendly дизайн

## 🚀 Автоматизация

### Фоновая задача (опционально)

Можно добавить периодическую проверку и автогенерацию:

```python
import asyncio
from apscheduler.schedulers.background import BackgroundScheduler

async def auto_generate_articles():
    """Автоматически генерирует статьи для новых популярных тем."""
    popular = article_generator.get_popular_topics(min_questions=5)
    
    for topic_data in popular:
        # Проверяем, не сгенерирована ли уже статья
        if not is_article_generated(topic_data["topic"]):
            await article_generator.generate_article_from_topic(
                topic_data=topic_data,
                language="ru",
                llm_manager=llm_manager
            )
            print(f"✅ Автогенерация: {topic_data['topic']}")

# Запускать каждый день в 3:00 AM
scheduler = BackgroundScheduler()
scheduler.add_job(auto_generate_articles, 'cron', hour=3, minute=0)
scheduler.start()
```

## 💡 Расширенные возможности

### 1. Агрегация лучших ответов

Система может выбирать лучшие ответы из истории диалогов:

```python
def get_best_answers(topic: str, tradition: str) -> List[str]:
    """Находит лучшие ответы по теме и традиции."""
    questions = [q for q in questions_db["questions"] 
                 if q["topic"] == topic and q["tradition_id"] == tradition]
    
    # Сортируем по длине (более подробные = лучше)
    questions.sort(key=lambda x: x["word_count"], reverse=True)
    
    return [q["answer"] for q in questions[:3]]
```

### 2. Сравнительный анализ

Автоматическое сравнение подходов разных традиций:

```python
async def generate_comparison(traditions: List[str], topic: str):
    """Генерирует сравнительную таблицу подходов."""
    prompt = f"""Сравни подходы к теме "{topic}" в следующих традициях:
    {', '.join(traditions)}
    
    Создай таблицу с колонками:
    - Традиция
    - Основной принцип
    - Практическая рекомендация
    - Цитата
    
    Верни в формате Markdown таблицы."""
    
    result = await llm_manager.generate_with_fallback(prompt=prompt)
```

```python
def extract_user_stories(topic: str) -> List[str]:

    stories = []
    
    for q in questions_db["questions"]:
        if q["topic"] == topic and len(q["question"]) > 50:
```

### 4. Динамическое обновление

Статьи можно обновлять при появлении новых вопросов:

```python
@app.post("/api/v1/articles/update/{topic}")
async def update_article(topic: str):
    """Обновляет существующую статью новыми данными."""
    # Проверяем новые вопросы с момента последней генерации
    new_questions = get_new_questions_since(topic, last_generated_date)
    
    if len(new_questions) >= 3:
        # Перегенерируем статью с новыми данными
        await regenerate_article(topic, include_new_questions=new_questions)
        return {"status": "updated", "new_questions_added": len(new_questions)}
    else:
        return {"status": "no_changes", "new_questions": len(new_questions)}
```
