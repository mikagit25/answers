
# 📘 ТЕХНИЧЕСКОЕ ЗАДАНИЕ: `Answers` (Web + PWA)

## 1. Общее описание
- **Название проекта:** Answers  
- **Цель:** Платформа, предоставляющая ответы на жизненные, этические и духовные вопросы в рамках выбранной пользователем религиозной или философской традиции, с обязательным цитированием первоисточников и возможностью сравнения взглядов.  
- **Целевая аудитория:** Рядовые русскоязычные пользователи, ищущие понятные, проверенные ответы без академического перегруза.  
- **Формат:** Адаптивный веб-интерфейс с поддержкой PWA (Progressive Web App). Работает в браузере, устанавливается на главный экран, кеширует источники для офлайн-доступа.  
- **Архитектурный принцип:** API-first. Единый бэкенд обслуживает веб, в будущем легко масштабируется на Telegram-бота, Android/iOS без переписывания ядра.  
- **Тон взаимодействия:** Нейтральный, уважительный, без догматических утверждений «от системы».

## 6. API контракт
### `POST /api/v1/ask`
**Request:**
```json
{
  "question": "Как преодолеть страх перед будущим?",
  "tradition_id": "stoicism",
  "depth": "basic"
}
```
**Response:**
```json
{
  "answer": "В стоицизме страх перед будущим преодолевается через...",
  "sources": [
    {
      "text": "«Люди страдают не от вещей, а от мнений о вещах»",
      "reference": "Эпиктет, Энхиридион, §5",
      "translation": "пер. С. Соболевского",
      "commentary": "Адаптация для современного читателя"
    }
  ],
  "tradition_id": "stoicism",
  "share_url": "/answer/stoicism-fear-future-a1b2c3",
  "metadata": { "model": "qwen-turbo", "tokens": 380, "cache_hit": false }
}
```

### `GET /api/v1/compare`
**Query:** `?question=...&exclude=stoicism`  
**Response:** `[{ tradition_id, summary, source_preview }]`

## 7. RAG-конвейер и валидация источников
1. **Чанкинг:** 300–500 токенов, перекрытие 50.
2. **Поиск:** Гибридный (векторный + BM25 по ID/ключевым словам).
3. **Генерация:** LLM получает только релевантные чанки + строгий системный промпт.
4. **Пост-валидатор:** Регулярные выражения + проверка по БД. Если `<2` подтверждённых ссылок → ответ помечается `{"status": "uncertain"}` и не показывается пользователю (или показывается с предупреждением).
5. **Кэширование:** Популярные запросы кешируются в Redis на 24ч с хэшем `sha256(question + tradition_id)`.

## 8. UI/UX & PWA требования
- **Главный экран:** Центрированный инпут, сверху горизонтальный скролл традиций, кнопка `Получить ответ`.
- **Карточка ответа:** Заголовок `Ответ согласно [Традиция]`, текст ≤ 400 слов, раскрывающийся блок источников, кнопки `🔍 Сравнить`, `📤 Поделиться`.
- **PWA:** 
  - `manifest.json` с `display: standalone`, `theme_color`, иконками
  - Service Worker кеширует: статику, `/api/v1/ask` (GET-кэш), базу источников
  - Fallback-страница при офлайне: список сохранённых ответов
  - Prompt установки появляется после 2 визитов или при скролле >60%
- **Адаптив:** На мобильных источники сворачиваются в аккордеон, шрифты ≥16px, отступы ≥8px.

## 12. Этика и безопасность
1. **Дисклеймер:** В футере ответа: `Ответ сформирован на основе открытых источников выбранной традиции. Не является богословским или профессиональным руководством.`
2. **Модерация:** Фильтрация запросов с призывами к насилию/дискриминации → вежливый отказ.
3. **Данные:** История хранится в `localStorage` по умолчанию. Религиозные предпочтения не передаются на сервер без явного согласия.
4. **Экспертная валидация:** Бета-тест с 2-3 консультантами по традициям перед публичным запуском.
